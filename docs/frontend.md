# フロントエンド設計 (Frontend)

本ディレクトリ (`src/frontend/`) および `src/app/` のページ部分に関する仕様です。

## 1. ページ構成 (Routing)

### `/` (ホーム)
- **機能**: ルーム作成、ログイン、ダッシュボードへの導線。

### `/rooms` (ダッシュボード)
- **機能**: 参加中・作成済みのルーム一覧、IDによる新規ルーム参加。

### `/rooms/[id]` (ルーム詳細 / キャラクター一覧)
- **機能**: ルーム内のキャラクター一覧表示（Dnd-kit による並び替え対応）、新規キャラクター追加、メンバーリスト。
- **UI**: キャラクター画像（1〜2枚）のグリッド表示。

### `/rooms/[id]/throw/[characterId]` (投下画面)
- **機能**: 特定のキャラクターへのブーケ投下。
- **演出**: **Bouquet Rain** によるブーケの蓄積演出。

### `/rooms/[id]/characters/[characterId]/stats` (統計ページ)
- **機能**: 贈られたブーケの詳細内訳、閲覧権限の管理。
- **UI**: **Animated Counter** によるカウントアップ演出。

### `/profile` (マイページ)
- **機能**: ユーザー情報の編集、自分が作成したキャラクター（騎士）の一覧確認。

## 2. コア・コンポーネント

### `KnightCard`
- キャラクターのビジュアルを担当。
- `avatarUrl` がカンマ区切りで2つある場合、自動的に `grid-cols-2` で左右分割表示します。

### `BouquetRain`
- `framer-motion` を使用したパーティクル演出。
- 投下されたブーケが画面下部にランダムな位置で積み上がる視覚効果を提供します。

### `LoadingScreen`
- 全ページ共通のプレミアムな読み込み画面。
- 花の浮遊アニメーションと、カスタマイズ可能なメッセージを表示します。

### `AnimatedCounter`
- 数値の増加をスムーズに表現するカウンター。

## 3. 状態管理と最適化

### バッチ送信ロジック (`useBouquetSender`)
- 連打されたクリックを `useRef` で蓄積し、一定時間（デバウンス）後にまとめて `POST /api/bouquets` を実行します。
- 楽観的更新により、ネットワーク遅延を感じさせない操作感を実現。

### リアルタイム同期 (SSE)
- 各ページで `EventSource` を使用し、サーバーからの `CHARACTER_UPDATE` などのイベントをリッスンして状態を自動更新します。

## 4. デザインシステム
- **テーマ**: Zinc (グレー) と Rose (ピンク) を基調とした、エレガントでプレミアムなデザイン。
- **タイポグラフィ**: `Outfit` (サンセリフ) と `Serif` フォントの組み合わせによる高級感の演出。
