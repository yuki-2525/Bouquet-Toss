# フロントエンド設計 (Frontend)

本ディレクトリ (`src/frontend/`) および `src/app/` のページ部分に関する仕様です。

## 1. ページ構成 (Routing)

Next.js App Router によって構成される各ページの役割と内容です。

### `/` (ルート: ホーム / ルーム作成・入室)
- **責務:** アプリケーションのエントリポイント。
- **機能:**
  - 既存のセッション（ルーム）への入室（ルームID/パスワード入力）。
  - 新規ルームの作成（ルーム名、パスワード、登場する騎士の設定）。
  - ユーザーログイン・サインアップへの導線（SNS連携等）。
- **UI:** ライト/ダークモードに対応したエレガントなポータル画面。

### `/rooms/[id]` (セッションルーム画面 / 一覧)
- **責務:** この部屋に登場するキャラクターの一覧を確認し、投下先を選択する画面。
- **機能:**
  - **キャラクター登録:** 参加ユーザーは、この部屋に登場する自分のキャラクター（騎士）を作成・追加できる。
  - **一覧表示:** キャラクターのリストを表示し、クリックで「ブーケ投下専用画面」へ遷移する。
- **UIコンポーネント:** `CharacterList`, `CharacterCreateModal`

### `/rooms/[id]/throw/[characterId]` (ブーケ投下専用画面)
- **責務:** 特定のキャラクターに対してブーケを全力で贈るための専用画面。
- **機能:**
  - **ターゲット表示:** 画面中央に対象のキャラクターを大きく表示。
  - **秘匿設計:** 画面に表示される数字は「自分がその騎士に贈った数」のみです。他の人が贈った数や総数は見えません。
  - **ブーケ投下ボタン:** 画面全体を覆うようなエフェクトと共に連打可能。押すたびに数字が増え、パーティクルが舞う。
- **UIコンポーネント:** `KnightCard`, `BouquetButton`, `ParticleEffect`

### `/rooms/[id]/characters/[characterId]` (オーナー用詳細・集計画面)
- **責務:** キャラクターが受け取ったブーケの総数などを確認する画面。
- **アクセス制限:** **このキャラクターを作成した本人（オーナー）しかアクセスできません。** 他のユーザーがURLを直接入力しても閲覧できません。
- **機能:**
  - 自分（のキャラクター）が受け取ったブーケの「総獲得数」の確認。
  - 誰からどれくらい貰ったかなどの詳細な内訳の表示。

### `/mypage` (マイページ / 統計)
- **責務:** ユーザー個人の活動履歴や統計を表示。
- **機能:**
  - 累計投下数、よくブーケを贈る相手（推し騎士）のランキング。
  - 過去に参加したセッションの履歴。
  - （※ゲストユーザーはアクセス不可、または正規登録への誘導を表示）

## 2. 状態管理と通信 (バッチ更新ロジック)

ボタン連打によるサーバー負荷を防ぐため、以下のロジックで通信を最適化します。

1. **楽観的UI更新:**
   ボタンをクリックした瞬間、サーバーからの応答を待たずに画面上のカウント（ローカルステート）を増やし、アニメーションを発火させます。
2. **ローカルバッファリング:**
   クリックされた回数を `useRef` を用いてメモリ上に蓄積します。
3. **Debounce (遅延) 送信:**
   連続したクリックが止まり、一定時間（例: 2秒）経過したタイミングで、バッファリングされた回数を1つのリクエストとしてAPI (`/api/bouquets`) に `POST` 送信します。
4. **状態の同期:**
   APIからのレスポンスを受け取り、必要であればサーバー側の正確なカウントで状態を同期します。

## 3. UI/UX コンポーネント設計

- **テーマ:** `next-themes` を利用し、漆黒(zinc-950)と深紅(rose-500)を基調としたダークモードと、視認性の高いライトモードの両方を提供。
- **アニメーション (`framer-motion`):**
  - ボタンホバー時のグローエフェクト。
  - クリック時の波紋(Ping)とボタンの押し込み(Scale)表現。
  - ブーケ（花びら）が舞い散るパーティクルエフェクト。
- **アイコン:** `lucide-react` を使用。
