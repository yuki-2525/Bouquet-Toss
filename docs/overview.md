# プロジェクト概要 (Overview)

## Bouquet-Toss プロジェクト概要

このプロジェクトは、TRPG『銀剣のステラナイツ』などのセッション中に、騎士や監督生に対してリアルタイムで「ブーケ（賞賛）」を贈り合うためのWebアプリケーションです。

## コア・コンセプト
- **「推し」を物理的に埋め尽くす**: 連打によってブーケが画面を埋め尽くす爽快な演出。
- **ペアの絆を可視化**: 騎士と監督生など、ペアで活動するキャラクターに同時にスポットライトを当てる。
- **献身の記録**: 誰がどれだけブーケを贈ったかを統計として残し、後で振り返ることができる。

## 主要機能

### 1. リアルタイム・ルームシステム
- Supabase Realtime を活用し、他者の投下を遅延なく全ユーザーの画面に反映。
- ドラッグ＆ドロップによるキャラクターの動的な並び替え（全ユーザーにリアルタイム同期）。

### 2. キャラクター・ビジュアル
- **マルチ・アバター対応**: 1キャラクターにつき最大2枚の画像を設定可能。
- **スマート・レイアウト**: 画像枚数に応じて「中央表示」「左右分割」が自動的に切り替わる。
- **カード型UI**: ルーム、投下画面、統計ページで一貫したプレミアムなカードデザインを採用。

### 3. 統計とアクセス管理
- **詳細内訳**: 各ユーザーがそのキャラクターに贈ったブーケ数をカウント。
- **閲覧制限**: キャラクターのオーナーは、統計ページの公開/非公開をユーザーごとに設定可能。
- **Animated Counter**: 獲得ブーケ数の増加をスムーズな数字アニメーションで表現。

### 4. プレミアム演出
- **Bouquet Rain**: 投げられたブーケが上から降り注ぎ、下部に蓄積される物理演算風の演出。
- **Loading Screen**: フローティングする花のアニメーションによるリッチな読み込み待機画面。
- **Glassmorphism**: モダンなグラスモルフィズムを取り入れたUIデザイン。

## 技術的特徴
- **バッチ更新**: フロントエンドでクリックをまとめ、サーバーへの負荷を軽減。
- **Edge Runtime**: OGP画像生成などに Edge Runtime を使用し、高速なレスポンスを実現。
- **Responsive**: PC、タブレット、スマートフォンすべてで最適化されたレイアウト。
- **バックエンド (BFF):** Next.js Route Handlers (`src/app/api`)
- **データベース・認証:** Supabase (PostgreSQL, Auth)
- **インフラ:** Vercel
