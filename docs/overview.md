# プロジェクト概要 (Overview)

## アプリケーションの目的
TRPG『銀剣のステラナイツ』のセッションで使用する、ブーケ（賞賛）をリアルタイムで贈るためのWebアプリケーションです。
「ボタンを連打して推しの騎士にブーケを浴びせる」爽快感と、プレイヤーの熱狂・献身を可視化することをコンセプトにしています。

## コア体験
- **爽快なアクション:** ボタン連打に即座に反応するエフェクト（パーティクル、波紋など）。
- **リアルタイム性:** セッション中、他の参加者が贈ったブーケも画面に反映されます。
- **記録と統計:** 「誰が、どのセッションで、誰にどれだけブーケを贈ったか」が記録され、後から振り返ることができます。

## 全体アーキテクチャ
Vercel へのデプロイを前提とし、フロントエンドとバックエンドの役割を1つの Next.js (App Router) リポジトリ内で明確に分離しています。

### 技術スタック
- **フロントエンド:** Next.js (React), Tailwind CSS, Framer Motion, next-themes
- **バックエンド (BFF):** Next.js Route Handlers (`src/app/api`)
- **データベース・認証:** Supabase (PostgreSQL, Auth)
- **インフラ:** Vercel

## ディレクトリ構造と責務
```text
Bouquet-Toss/
├── docs/                       # ドキュメント群
├── src/
│   ├── app/                    # Next.js App Router (ページ、ルーティング、API)
│   │   └── api/                # 【バックエンドAPI】Vercel Serverless Functions
│   │
│   ├── frontend/               # 【フロントエンド層】UIコンポーネント、フック、通信
│   │
│   └── backend/                # 【バックエンド層】DB操作、Supabase設定、サーバーロジック
```
